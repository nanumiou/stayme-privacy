개인정보처리방침 (Privacy Policy)

Last Updated: March 12, 2026

1. 데이터 처리 원칙

StayMe는 사용자의 프라이버시를 최우선으로 설계되었습니다. 본 앱은 사용자를 추적하거나 개인정보를 수집하여 서버에 영구적으로 보관하지 않습니다. 모든 서비스는 서비스 본연의 가치인 '실시간성'과 '익명성'을 바탕으로 운영됩니다.

2. 데이터 수집 및 보관 방식

본 앱은 데이터를 두 가지 방식으로 명확히 구분하여 처리합니다.

A. 기기 내 로컬 저장 (Local Storage)

다음 정보는 서버로 전송되지 않으며, 오직 사용자의 기기에만 저장됩니다. 앱을 삭제하면 함께 제거됩니다.

  • 개인 설정 (타이머 시간, 소리 설정, 테마 모드 등)
  • 사용자 닉네임 (기기 내 저장용)
  • 온보딩 완료 여부 및 선호하는 동반자 타입

B. 실시간 전송 데이터 (Realtime Presence)

실시간 접속자 표시 기능을 위해 일시적으로 서버(Firebase)를 통과하거나 머무는 데이터입니다.

  • 익명 UID: 무작위 생성된 식별값으로, 실제 인적 사항과 연결되지 않습니다.
  • 실시간 상태: 현재 접속 여부, 닉네임, 접속 시작 시각이 포함됩니다.

3. 데이터 파기 정책 (Zero-Retention)

StayMe는 사용자가 앱을 종료하거나 네트워크 연결이 끊기는 즉시, 서버에 기록된 접속 상태 데이터를 자동 파기(Auto-Remove)하도록 설계되어 있습니다.

본 앱은 사용자의 과거 집중 기록이나 로그를 서버상의 데이터베이스에 누적하여 저장하지 않습니다. 따라서 별도의 삭제 요청 절차 없이도 사용자가 앱을 사용하지 않는 순간 데이터는 서버에서 완전히 사라집니다.

4. 이용 목적 및 제3자 제공

수집된 데이터는 '실시간 집중자 공유'라는 본연의 목적을 위해서만 사용됩니다. StayMe는 광고 목적이나 마케팅을 위해 데이터를 제3자에게 판매하거나 공유하지 않습니다.

  • 서비스 인프라: 안정적인 데이터 처리를 위해 Google Firebase 인프라를 사용하며, 이는 구글의 보안 표준을 따릅니다.

5. 사용자의 권리

사용자는 앱 내 설정에서 언제든지 닉네임을 변경하거나 실시간 접속자 표시 기능을 제어할 수 있습니다. 수집되는 데이터 자체가 휘발성이므로, 앱을 삭제하는 것만으로 모든 데이터 처리를 즉시 중단할 수 있습니다.
