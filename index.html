개인정보처리방침 (Privacy Policy)

Last Updated: April 23, 2026

1. 데이터 처리 원칙

StayMe는 사용자의 프라이버시를 최우선으로 설계되었습니다. 기본적으로 앱은 '익명성'을 바탕으로 운영되며, 실시간 집중에 필요한 데이터만 일시적으로 처리합니다. 단, 'StayMe Premium' 구독 서비스를 이용하기 위해 자발적으로 소셜 계정을 연동한 사용자에 한하여 결제 및 권한 유지를 위한 최소한의 데이터를 보관합니다.

2. 데이터 수집 및 보관 방식

본 앱은 사용 형태에 따라 데이터를 세 가지 방식으로 구분하여 처리합니다.

A. 기기 내 로컬 저장 (모든 사용자)

다음 정보는 서버로 전송되지 않으며, 오직 사용자의 기기에만 저장됩니다. 앱을 삭제하면 함께 제거됩니다.

  • 개인 설정 (타이머 시간, 배경화면, 테마 모드 등)
  • 사용자 기기 전용 닉네임
  • 온보딩 완료 여부

B. 실시간 전송 데이터 (모든 사용자)

실시간 접속자 표시 기능을 위해 일시적으로 서버를 통과하거나 머무는 휘발성 데이터입니다.

  • 익명 접속 ID: 무작위 생성된 식별값으로, 단독으로 실제 인적 사항을 특정할 수 없습니다.
  • 실시간 상태: 현재 접속 여부, 설정된 닉네임, 접속 시작 시각이 포함되며 연결이 끊기면 파기됩니다.

C. 프리미엄 구독자 데이터 (구독 사용자 한정)

프리미엄 구독을 위해 Apple 또는 Google 소셜 로그인을 연동한 사용자에 한해 다음 정보를 보관합니다.

  • 보안 식별자: 타 기기 로그인 및 구독 복원을 위한 암호화된 Provider UID (이메일 등은 앱에서 명시적으로 표시하거나 마케팅에 사용하지 않습니다).
  • 결제 상태 정보: 현재 구독 유효 여부(isPremium) 및 만료 일자(expiryDate). 본 앱은 신용카드 번호 등 민감한 금융 정보를 직접 수집하거나 처리하지 않습니다.

D. 그룹 채팅 데이터 (서비스 이용자 한정)

동일한 초대 코드를 공유하는 '그룹 모드' 사용 시 원활한 소통을 위해 다음 정보를 일시적으로 보관합니다.

  • 채팅 메시지: 사용자가 직접 입력한 텍스트 내용 및 전송 시각.
  • 발신자 정보: 메시지에 표시될 닉네임 및 사용자 식별 ID(UID).
  • 공개 범위: 동일한 초대 코드를 입력하여 해당 그룹에 참여한 모든 멤버에게 공개됩니다.

3. 데이터 보관 기간 및 파기

익명 사용자는 앱을 삭제하는 것만으로 기기에 저장된 모든 로컬 데이터가 삭제되며, 서버에 기록된 접속 상태 데이터는 즉시 자동 파기(Zero-Retention)됩니다.

그룹 채팅 메시지의 경우, 대화의 연속성을 위해 서버에 최대 24시간 동안 보관된 후 자동으로 영구 삭제됩니다.

소셜 계정을 연동한 사용자의 경우, 앱 내 '설정 > 계정 관리' 메뉴의 [계정 삭제] 버튼을 통해 언제든지 직접 탈퇴를 진행할 수 있습니다. 탈퇴 시 관련된 모든 사용자 식별자 및 서버 데이터는 즉시 영구 삭제됩니다.

4. 웹(Web)을 통한 계정 삭제 요청

앱을 이미 삭제했거나 앱에 접속할 수 없는 경우, 아래의 방법으로 계정 및 관련 데이터 삭제를 요청하실 수 있습니다.

  • 요청 방법: 지원 이메일로 계정 식별 정보(소셜 로그인 이메일 주소 등)를 기재하여 발송
  • 이메일 주소: support@franholdings.cloud
  • 처리 기간: 요청 접수 후 영업일 기준 7일 이내 완료 및 확인 회신

※ 주의: 계정 삭제 시 프리미엄 멤버십 정보 및 모든 활동 데이터가 영구히 삭제되며 복구할 수 없습니다. 구독 관리(해지)는 각 스토어 계정 설정에서 직접 진행하셔야 합니다.

5. 제3자 서비스 제공 및 위탁 (Third-Party Services)

안정적인 기능 제공과 정기 결제 확인을 위해 다음의 공신력 있는 외부 인프라를 사용하고 있습니다. 데이터는 해당 목적 이외(광고, 제3자 마케팅 등)로 절대 사용되지 않습니다.

  • Google Firebase (Analytics): 앱 이용 행태 분석, 기능별 선호도 측정 및 서비스 개선을 위한 통계 데이터 수집 (익명화된 앱 인스턴스 식별자 기반).
  • Google Firebase (Auth/Database): 실시간 데이터 바인딩 및 소셜 로그인 인증을 위한 암호화된 클라우드 인프라 제공.
  • RevenueCat: 앱스토어 및 플레이스토어 인앱 결제 영수증(Receipt) 검증 및 구독 생애주기 관리 (익명화된 App User ID 기반으로 통신).

6. 사용자의 권리

사용자는 닉네임과 실시간 기능을 언제든 제어할 수 있습니다. 수집되는 데이터 대부분이 휘발성이며, 익명 사용자는 앱을 삭제하는 것만으로 데이터 처리를 중단할 수 있습니다. 소셜 연동 사용자의 경우, 앱 내 '계정 삭제' 기능 또는 위의 웹 삭제 요청을 통해 즉시 데이터를 파기할 수 있습니다.

7. 문의처

프라이버시와 데이터 처리, 혹은 계정 삭제와 관련하여 문의사항이 있으시면 아래로 연락 부탁드립니다.

StayMe

Privacy Policy

Last Updated: April 23, 2026

1. Principles of Data Processing

StayMe is designed with user privacy as its top priority. By default, the app operates based on **"Anonymity"** and only temporarily processes data required for real-time focus. However, for users who voluntarily link their social accounts to use the **"StayMe Premium" subscription service**, we retain the minimum necessary data to maintain payment and access rights.

2. Data Collection and Storage Methods

This app processes data in three ways depending on the usage pattern.

A. Local Storage on Device (All Users)

The following information is not transmitted to the server and is stored only on the user's device. It is removed when the app is deleted.

  • Personal settings (Timer duration, wallpaper, theme mode, etc.)
  • Device-specific nickname
  • Onboarding completion status

B. Real-time Transmission Data (All Users)

This is volatile data that temporarily passes through or stays on the server to provide the real-time active user display feature.

  • Anonymous Session ID: A randomly generated identifier that cannot be used on its own to identify a specific individual.
  • Real-time Status: Includes current connection status, set nickname, and connection start time. This data is destroyed when the connection is lost.

C. Premium Subscriber Data (Subscribed Users Only)

For users who link their Apple or Google social accounts for a premium subscription, the following information is retained.

  • Security Identifier: Encrypted Provider UID for logging in on other devices and restoring subscriptions (Email addresses, etc., are not explicitly displayed in the app or used for marketing).
  • Payment Status Information: Current subscription validity (isPremium) and expiration date (expiryDate). This app does not directly collect or process sensitive financial information such as credit card numbers.

D. Group Chat Data (Service Users Only)

When using the 'Group Mode' sharing the same invitation code, the following information is temporarily stored for smooth communication.

  • Chat Messages: Text content and transmission time manually entered by the user.
  • Sender Information: Nickname to be displayed in messages and User Identification ID (UID).
  • Disclosure Scope: Disclosed to all members who have entered the same invitation code and joined the group.

3. Data Retention Period and Destruction

For anonymous users, all local data stored on the device is deleted simply by deleting the app, and connection status data recorded on the server is immediately and automatically destroyed (Zero-Retention).

For **group chat messages**, they are stored on the server for a **maximum of 24 hours** for conversation continuity and then automatically and permanently deleted.

For users who have linked a social account, they can withdraw at any time via the **[Delete Account] button in the 'Settings > Account Management' menu** within the app. Upon withdrawal, all associated user identifiers and server data are immediately and permanently deleted.

4. Account Deletion Request via Web

If you have already deleted the app or cannot access it, you can request the deletion of your account and related data using the method below.

  • Request Method: Send a support email including account identification information (social login email address, etc.).
  • Email Address: support@franholdings.cloud
  • Processing Time: Completed within 7 business days after receipt of the request, with a confirmation reply sent.

※ Caution: Upon account deletion, premium membership information and all activity data will be permanently deleted and cannot be recovered. Subscription management (cancellation) must be handled directly in the respective store account settings.

5. Third-Party Services

We use the following reputable external infrastructures to provide stable functionality and verify recurring payments. Data is strictly never used for purposes other than these (advertising, third-party marketing, etc.).

  • Google Firebase (Analytics): Collects statistical data for app usage behavior analysis, preference measurement by feature, and service improvement (based on anonymized app instance identifiers).
  • Google Firebase (Auth/Database): Provides encrypted cloud infrastructure for real-time data binding and social login authentication.
  • RevenueCat: Verifies App Store and Play Store in-app purchase receipts and manages subscription lifecycles (communicates based on anonymized App User IDs).

6. User Rights

Users can control their nickname and real-time features at any time. Most of the collected data is volatile, and anonymous users can stop data processing simply by deleting the app. For social-linked users, data can be destroyed immediately via the **'Delete Account' feature within the app** or the **web deletion request** mentioned above.

7. Contact Information

If you have any questions regarding privacy, data processing, or account deletion, please contact us below.

Support Team: support@franholdings.cloud
